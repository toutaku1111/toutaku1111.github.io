# 3x3x4X Card Set Draft

ローグライク風カードシステム開発用の仕様メモとカード効果サンプルです。

## カードシステム仕様

### カード入手

- プレイヤーが土地を手に入れた時、カード候補3枚から1枚を選んで入手する。
- カードを入手した時点で、そのカードは取得した土地の特性として紐づく。
- カード入手の選択は、すべてのフェーズ終了時点、次のターンへ移る直前に行う。
- カードは最大2枚まで保持できる。
- すでにカードを2枚持っている場合、新しいカードは入手できない。
- 相手の土地を奪った場合、以下のどちらかを選べる。
  - 既存の土地効果をそのまま引き継ぐ。この場合、カード入手はない。
  - 既存の土地効果を破棄し、新規3択からカードを1枚選ぶ。
- カードと土地特性はお互いに公開情報とする。

### 土地特性

- 入手したカードは、紐づいた土地の特性を決める。
- 土地特性は、その土地が2P以上になった時点で発動条件を満たす。
- 土地効果は、資産ゲットフェーズの前に解決する。
- 土地が1Pに下がった場合、その土地特性は保持されるが、2P以上に戻るまで発動しない。
- 要塞化などで資産を生まない土地でも、2P以上であれば土地特性の発動対象になり得る。最終的な扱いは実装時にルールで固定する。

### カード利用

- カード利用は1ターンに1枚まで。
- カード利用はコマンド指定の前に行う。
- このカード利用タイミングも、両プレイヤーの同時入力とする。
- カード利用後に通常のコマンド指定へ進む。
- 後先を厳密に参照しないと成立しない効果は、カードデザインとして不正とする。
  - 例: 「相手がこのターン侵攻していた場合だけ無効化する」
  - 例: 「相手が先に中央を取った場合だけ発動する」
- 同時入力で処理しても解釈がぶれない効果を基本とする。

## カード項目

| 項目 | 内容 |
| --- | --- |
| ID | カードを一意に識別するID |
| 名称 | 画面表示用のカード名。土地に根差した施設・地形・拠点名を基本にする |
| フレーバー | その土地がどんな場所になったのかを短く示す説明文 |
| カード利用したときの効果 | 手札からカードを使用した時に発生する能動効果 |
| 引いたときの土地効果 | カード入手時に土地へ紐づく特性。土地が2P以上なら資産ゲットフェーズ前に発動 |
| 使用コスト | カード使用時に必要な資産など |
| 参照するアセットNo | 表示画像・アイコン・演出素材の参照番号 |

## デザインメモ

- カード利用効果は、同時入力で処理順の解釈が割れないものにする。
- 土地効果は、原則として資産ゲットフェーズ前に一括解決できるものにする。
- 「次に」「最初の」などを使う場合は、解決順をカード仕様側で明確にする。
- 相手の未公開選択を直接参照する効果は避ける。
- 公開情報として扱うため、手札・土地特性・紐づき先の土地はUIで常時確認できる必要がある。
- カード名とフレーバーは、土地に何が建ったか、土地がどんな地形になったかが一目で伝わるものを優先する。
